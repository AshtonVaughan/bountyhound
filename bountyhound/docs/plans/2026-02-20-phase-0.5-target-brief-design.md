# Phase 0.5 Target Brief â€” Design Document

**Date:** 2026-02-20

## Goal

Add a pre-recon intelligence phase to bountyhound-agent that researches a target before a single packet is sent. Output a structured `TargetBrief` consumed by the Discovery Engine to generate targeted (not generic) hypotheses.

## Problem

The Discovery Engine currently generates hypotheses from scratch using only the stack detected during recon. It has no knowledge of:
- What vulnerabilities have already been found in this program (H1 disclosed reports)
- Known CVEs affecting the target's software versions
- Recent security patches the target has shipped (which may have introduced regressions or edge cases)

joaxcar's edge: he reads patch notes and changelogs before testing. Phase 0.5 automates this.

---

## Architecture

Phase 0.5 splits into two sub-phases to respect data availability:

```
Phase 0.5a  (BEFORE recon)   H1 disclosed reports + GitHub changelog
Phase 1     (recon)          subfinder + httpx â†’ stack detected
Phase 0.5b  (AFTER recon)    CVE lookup using detected stack
Phase 2+    (testing)        Discovery Engine receives full TargetBrief
```

---

## Data Sources

### 1. H1 Fetcher (`engine/intel/h1_fetcher.py`)
- **API:** `https://hackerone.com/graphql` (public, no auth)
- **Query:** Get disclosed reports for program handle, last 50 reports
- **Extracts:** vuln title, CWE type, affected endpoint, bounty amount, report date
- **Cache TTL:** 24 hours at `~/.bountyhound/intel/<program_handle>_h1.json`

### 2. Changelog Fetcher (`engine/intel/changelog_fetcher.py`)
- **API:** GitHub search API + raw file fetches
- **Logic:** Search GitHub for repos matching company name â†’ fetch `/releases` page or `CHANGELOG.md` / `CHANGELOG.rst`
- **Filter:** Regex match lines containing: `fix`, `patch`, `auth`, `bypass`, `injection`, `xss`, `csrf`, `vuln`, `security`, `CVE`
- **Output:** List of matching lines from last 90 days
- **Cache TTL:** 24 hours

### 3. CVE Fetcher (`engine/intel/cve_fetcher.py`)
- **API:** NVD `https://services.nvd.nist.gov/rest/json/cves/2.0?keywordSearch=<stack_token>`
- **Timing:** Runs AFTER Phase 1 recon so real stack tokens are available
- **Filter:** Last 3 years, CVSS â‰Ą 7.0 only
- **Output:** List of `{id, description, cvss, affected_versions}`
- **Cache TTL:** 24 hours per stack token

## Graceful Degradation

Each fetcher is independent. Failures are logged and skipped:
- H1 returns no results (private program, no disclosed reports) â†’ `disclosed_vulns: []`
- NVD API down â†’ `known_cves: []`
- No GitHub repo found â†’ `recent_changes: []`
- All fetchers fail â†’ `TargetBrief` is empty, hunt proceeds as today with no regression

---

## Caching

All fetcher outputs cached to `~/.bountyhound/intel/`:
- `<program_handle>_h1.json`
- `<program_handle>_cves.json`
- `<program_handle>_changelog.json`

Cache includes `generated_at` timestamp. Stale check: `(now - generated_at) > 86400s`.
